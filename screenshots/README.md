## Screenshot Guidelines

### Resolution and Quality
- Use 1920x1080 resolution for desktop screenshots
- Use actual mobile device or responsive mode for mobile screenshots
- Save as PNG format for best quality
- Ensure text is readable and UI elements are clear

### Content Guidelines
- Use sample data that looks realistic but not real personal information
- Show the application in a good state (no errors, proper loading)
- Include diverse question types and content
- Demonstrate key features and functionality

### Naming Convention
- Use lowercase with hyphens for file names
- Be descriptive but concise
- Match the names referenced in README.md exactly

## Taking Screenshots

### Desktop Screenshots
1. Start both frontend (port 8080) and backend (port 3001)
2. Navigate to http://localhost:8080
3. Create test accounts for different user types
4. Navigate through each mode and feature
5. Use browser developer tools to simulate mobile views
6. Take screenshots using your preferred tool (Snipping Tool, etc.)

### Mobile Screenshots
1. Use browser responsive mode or actual mobile device
2. Test portrait and landscape orientations
3. Ensure touch targets are visible and properly sized
4. Capture key interactions and gestures
